# Quickstart: Evolution API + BillionMail + Portainer

## Prerequisites

- Docker and Docker Compose installed
- Host PostgreSQL running on port 5432
- Ports 25, 80, 443, 587, 993, 8089, 8090, 9000 available
- DNS configured: `noaman.cloud` (A record), `mail.noaman.cloud` (A record), MX record

## 1. Create Evolution API database

```bash
sudo -u postgres psql -c "CREATE USER evolution_user WITH PASSWORD '<password>';"
sudo -u postgres psql -c "CREATE DATABASE evolutiondb OWNER evolution_user;"
```

## 2. Configure environment

```bash
# Copy example env files
cp .env.example .env

# Edit .env with your values:
# - EVOLUTION_API_KEY (generate a secure random key)
# - EVOLUTION_DATABASE_URI (postgresql://evolution_user:<password>@host.docker.internal:5432/evolutiondb)
# - BILLIONMAIL_HOSTNAME (noaman.cloud)
# - BILLIONMAIL_ADMIN_PASSWORD
# - BILLIONMAIL_DB_PASS
```

## 3. Start all services

```bash
# From project root
docker compose up -d
```

This starts: api-scraper, API-CR, Portal, scrape-sw-gsheet, Evolution API, Evolution Redis, BillionMail (7 containers), Portainer.

## 4. Initialize BillionMail

On first run, BillionMail needs database initialization:

```bash
# Run BillionMail's init script inside the core container
docker exec core-billionmail /opt/init-db.sh
```

## 5. Verify services

```bash
# Evolution API
curl http://localhost:8089/health

# BillionMail admin
curl http://localhost:8090/

# Portainer
curl http://localhost:9000/

# Existing services (should still work)
curl http://localhost:8080/health  # api-scraper
curl http://localhost:8086/health  # API-CR
curl http://localhost:8082/health  # Portal
```

## 6. Configure DNS (production)

Add these DNS records for `noaman.cloud`:

| Type | Name | Value |
|------|------|-------|
| A | `mail.noaman.cloud` | `<VPS_IP>` |
| MX | `noaman.cloud` | `mail.noaman.cloud` (priority 10) |
| TXT | `noaman.cloud` | `v=spf1 a mx ip4:<VPS_IP> -all` |
| TXT | `_dmarc.noaman.cloud` | `v=DMARC1; p=quarantine` |

DKIM key is generated by BillionMail's Rspamd — retrieve it after first start.

## 7. Access services

| Service | URL | Auth |
|---------|-----|------|
| Evolution API | `noaman.cloud/evolution/` | API key in header |
| BillionMail Admin | `mail.noaman.cloud/<SafePath>` | Username/password |
| RoundCube Webmail | `mail.noaman.cloud/roundcube/` | Email/password |
| Portainer | `noaman.cloud/portainer/` | Username/password (set on first access) |
