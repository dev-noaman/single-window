# Generate an OAuth 2.0 token

Source: https://developer.officernd.com/reference/post_generate-token

> ❗️
>
> ### Important   [Skip link to Important](https://developer.officernd.com/reference/post_generate-token\#important)
>
> This token is used as authorisation for most of the endpoints in the OfficeRnD API.
>
> The [User API](https://developer.officernd.com/reference/put_password_change) has requests that require a token generated by the [Signin API](https://developer.officernd.com/reference/post_auth-signin).

> ❗️
>
> ### Important   [Skip link to Important](https://developer.officernd.com/reference/post_generate-token\#important-1)
>
> This endpoint has a rate limit of 5 requests per minute, regardless if failed or successful. Once the limit is reached, error 429 with message "Too many requests, please try again later" will be returned.

client\_id

string

required

The client\_id of your application.

client\_secret

string

required

The client\_secret of your application.

grant\_type

string

required

When generating an OAuth 2.0 token the grant\_type should **always** be set to "client\_credentials".

scope

string

required

The scope of your application.

# `` 200      Token was generated successfully.

object

token

string

The token which can be used for authentication.

token\_type

string

The token type.

expires\_in

string

The remaining time until the token expires in seconds. The token always expires after an hour.

scope

string

The token scope.

# `` 400      Invalid client\_id, client\_secret, credentials or scope.
